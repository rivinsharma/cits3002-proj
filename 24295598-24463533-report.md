## Introduction
The aim of this project was to implement a simplified network simulator that demonstrates how Layer 2, Layer 3, and
Layer 4 work together and communicate to deliver data from one host to another.

The simulator then models how the two hosts connected through a router communicate while implementing simplified versions of the Transport, Network, and Data Link Layer functionalities.

The projects main focus was to demonstrate the core concepts that are involved in the communication and data transfer between these layers.

## Design Overview

Our design of the simulator was design with these four files
1. main.py
2. devices.py
3. protocol.py
4. config.py

For more information on what each file does specifically, please refer to our README.md.
The implementation seperates the protocols data structures from device behaviour to enhance its modularity and readability.

## Transport layer Design
The responsibilities of this layer include:
- Segment creation
- Sequence numbering
- Checksum handling
- ACK generation
- Retransmissions
- Segmentation

In this layer, we implemented an alternating bit protocol, whereby the sequence number would alternate from 0 to 1 or 1 to 0 after a correct ACK is received.
Typically a sender waits for the correct ACK before then continuing its transmission.
If a situation occurs whereby an ACK is not received or an incorrect sequence number is detected the segment is retransmitted.

## Segmentation Design
One of the requirements is that large messages should be segmented. This was done in this line:
```python
segments = [data[i:i+MAX_SEGMENT_DATA] for i in range(0, len(data), MAX_SEGMENT_DATA)]
```
This ensure the application data is split into chunks of 500 bytes, and/or its remaining bytes.

## Checksum Design
Checksums are then genereated inside the Segment class.
The checksum combines:
- Source port
- Destination port
- Sequence number
- Data length

The checksum is then recomputed at the receiver and compared against the received checksum.
If this verification step fails, the segment is then discarded.

## Network Layer Design
The responsibilities of this layer include:
- Packet creation
- Routing
- TTL decrementing
- Packet forwarding
Our design ensured that packets are encapsulated with source and destination IP addresses.
Routers then decrement the TTLE before forwarding.
If TTL reaches zero, the packet is then discarded.

## Routing Table Design
Our design ensure that the router accesses its routing table to determine its forwarding behaviour.
The routing table maps destination subnets to:
- Outgoing interfaces
- Next hop destinations

The router is able to perform route lookups using:
```python
lookup_route(self, destination_ip)
```
This reduces the use of hardcode and allows forwarding decisions to be driven by configuration instead.

## Data Link Layer Design
The responsibilities of this layer include:
- Frame creation
- MAC addressing
- MAC lookup
- Frame forwarding

Frames then encapsulate packets via source and destination MAC addresses.
Hosts and routers then use MAC tables to determine next hop MAC addresses.

## Reliable Data Transfer
In our design, we achieved reliable data transfer by using:
- ACK packets
- Sequence numbers
- Retransmissions
- Checksum verification
Transmission is only continued by the sender if the correct ACK is received. This is to ensure our ordered and reliable delivery.

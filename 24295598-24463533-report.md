## Introduction
The aim of this project was to implement a simplified network simulator that demonstrates how Layer 2, Layer 3, and
Layer 4 work together and communicate to deliver data from one host to another.

The simulator models how the two hosts which are connected through a router communicate with each other, whilst also including the implementation of simplified versions of the Transport, Network, and Data Link Layer functionalities.

The project's main focus was to demonstrate the core concepts that are involved in the communication and data transfer between these layers.

## Design Overview

The design of the simulator was implemented through these four files:
1. main.py
2. devices.py
3. protocol.py
4. config.py

For more information on what each file specifically does, please refer to the README.md.

The implementation seperates the protocol's data structures, from device behaviour, to enhance its modularity and readability.

## Transport layer Design
The responsibilities of this layer include:
- Segment creation
- Sequence numbering
- Checksum handling
- ACK generation
- Retransmissions
- Segmentation

In this layer, an alternating bit protocol was implemented, whereby the sequence number would alternate from 0 to 1 or 1 to 0 after a correct ACK is received as per the project requirements. 

Typically a sender waits for the correct ACK to be received before continuing its transmission of the following segments.

If a situation occurs whereby an ACK is not received or an incorrect sequence number is detected the segment is retransmitted.

## Segmentation Design
According to the project requirments, messages with data sizes above 500 were required to be segmented. 

Within the devices.py file, this requirment was addressed in the following line:
```python
segments = [data[i:i+MAX_SEGMENT_DATA] for i in range(0, len(data), MAX_SEGMENT_DATA)]
```
This line initialises a list containing the segments to be transmitted, whilst adhering to maximum segment size of 500. 
As a reult, this ensures that the application data is split into chunks of 500 bytes, and/or its remaining bytes (for the final segment).

## Checksum Design
Checksums are then genereated within the Segment class (protocol.py file).
The checksum combines:
- Source port
- Destination port
- Sequence number
- Data length

The checksum is then recomputed at the receiver and compared against the received checksum.
```python
if segment.checksum != segment.compute_checksum():
```
If this verification step fails, the segment is then discarded.

## Network Layer Design
The responsibilities of this layer include:
- Packet creation
- Routing
- TTL decrementing
- Packet forwarding

The design implemented ensures that packets are encapsulated with source and destination IP addresses.
```python
packet = Packet(
                    src_ip=self.ip,
                    dst_ip=destination_ip,
                    payload=segment
                )
```

Routers then decrement the TTL before forwarding.
```python
old_ttl = packet.ttl
        packet.ttl -= 1
```

If TTL reaches zero, the packet is then discarded.
```python
if packet.ttl <= 0:
            print(f"{self.name}: Layer 3: TTL expired. Packet discarded \n")
            return
```

## Routing Table Design
This design ensures that the router accesses its routing table to determine its forwarding behaviour.
The routing table maps destination subnets to:
- Outgoing interfaces
- Next hop destinations

```python
routing_table={
        "10.0.1.0/24": {
            "interface": "Interface 1",
            "next_hop": HOST_A_IP
        },
        "10.0.2.0/24": {
            "interface": "Interface 2",
            "next_hop": HOST_B_IP
        }
    }
```

The router is able to perform route lookups using:
```python
lookup_route(self, destination_ip)
```
This reduces the use of hardcoding within implementation of the simulator and allows forwarding decisions to be driven by configuration instead.

## Data Link Layer Design
The responsibilities of this layer include:
- Frame creation
- MAC addressing
- MAC lookup
- Frame forwarding

Frames encapsulate packets via source and destination MAC addresses.
```python
frame = Frame(
                    src_mac=self.mac,
                    dst_mac=next_hop_mac,
                    payload=packet
                )
```

Hosts and routers then use MAC tables to determine the next hop MAC addresses.
```python
next_hop_mac = self.mac_table[self.default_gateway_ip]
```
